# Task 5 — Build the dashboard UI

Create the Version History Dashboard interface.

Requirements:
- Show a timeline of versions.
- Display version label, status, author, and timestamp.
- Provide actions to inspect, compare, and restore versions where allowed.
- Keep Indonesian as the default UI language and English support intact.

Acceptance criteria:
- Users can browse history clearly.
- The UI matches the app’s existing style and flow.
- Prompt and content histories are visually separated.
