# Task 3 — Build history APIs

Create backend endpoints for listing version history and restoring versions.

Requirements:
- Provide endpoints for prompt history and content history.
- Support listing, viewing detail, and comparing versions.
- Add restore or rollback behavior only where it is safe.
- Do not silently overwrite history.

Acceptance criteria:
- History can be fetched from the API.
- Restore requests are validated.
- Unauthorized access remains blocked.
