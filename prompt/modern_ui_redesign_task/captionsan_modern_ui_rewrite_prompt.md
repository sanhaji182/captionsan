# CaptionSan — Modern UI Rewrite Prompt

Redesign CaptionSan into a modern, clean, and discoverable product.

## Product goals
- Make the app feel premium, calm, and easy to explore.
- Keep Indonesian as the default UI language and English supported.
- Preserve the prompt-first workflow.
- Make review, templates, history, and automation easier to understand.
- Avoid the rigid, cluttered, or overly technical feel.

## Design requirements
- Light mode first.
- Spacious layout with clear hierarchy.
- One obvious next action on each main screen.
- Strong empty states and contextual onboarding.
- Progressive disclosure for advanced features.
- Subtle motion only.
- Consistent component system with clear spacing, radius, and typography.

## Key screens to redesign
- Home dashboard.
- Prompt creation flow.
- Template library.
- Review and approval screen.
- Version history screen.
- Automation / n8n sync status screen.

## Experience requirements
- New users should know what to do within seconds.
- Users should discover features without needing a manual.
- The interface should feel cohesive across pages.
- Status, next actions, and context should always be visible.

## Implementation approach
1. Review the existing UI structure.
2. Apply a modern design system and layout hierarchy.
3. Redesign dashboard and main workflow screens first.
4. Add onboarding, empty states, and helper text.
5. Refine components, motion, and accessibility.
6. Validate that bilingual text and prompt-first flow still work.

## Output expected from you
When finished, summarize:
- What was redesigned.
- What files changed.
- Any assumptions or blockers.
- Whether the redesign is complete or still needs follow-up.
