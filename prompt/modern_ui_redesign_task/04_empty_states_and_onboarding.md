# Task 4 — Empty States and Onboarding

Redesign empty states and onboarding so feature discovery feels natural.

Requirements:
- Add useful empty states on all major pages.
- Make onboarding lightweight, skippable, and contextual.
- Avoid forcing long tours or intrusive modals.
- Keep the UI helpful from the first visit.

Acceptance criteria:
- Empty states guide users toward action.
- Onboarding helps without feeling heavy.
- Discovery is clear and non-blocking.
