# Task 3 — Version History Dashboard

Build the Version History Dashboard for prompts and generated content.

Requirements:
- Show prompt versions and content versions separately.
- Provide timeline, author, timestamp, status, and version label.
- Support compare and restore actions where safe.
- Keep the UI bilingual and consistent with existing design.

Acceptance criteria:
- Users can browse history clearly.
- Differences between versions are inspectable.
- Restore actions do not break current records.
